# 📋 离线问卷二维码传输系统

> 患者填写问卷 → 编码压缩 → 生成二维码 → 医院扫码 → 解码还原

一套**完全离线**的医疗问卷数据传输方案。患者在候诊时填写电子问卷，答案经过编码压缩后生成二维码，医生扫码即可还原全部数据——全程无需联网。

## ✨ 核心特性

### 问卷传输
- 🔒 **完全离线** — 数据通过二维码传输，无需网络
- 📦 **高效压缩** — 自定义编码 + LZ-String 压缩，70-80% 压缩率
- 📊 **实时反馈** — 填写时实时显示体积、容量预警、字数限制
- 📱 **多端适配** — 响应式设计，支持手机和电脑
- 🎯 **智能编码** — 多选题自动选择最优编码策略（列表/位图/组合映射）

### 🆕 文件传输
- 📁 **任意文件** — 支持图片、文档、音频、视频等所有文件类型
- 🔗 **Base64 编码** — 文件 → Base64 → 压缩 → 分片
- 📡 **连续扫描** — 自动识别并收集分片，无需手动操作
- ✅ **完整性校验** — CRC32 校验确保数据传输准确
- 💾 **一键下载** — 重组后直接下载原始文件

## 🚀 快速开始

### 环境要求

- Node.js 20+
- pnpm（推荐）或 npm

### 安装 & 启动

```bash
# 克隆项目
git clone <repo-url>
cd offline-questionnaire-qrcode

# 安装依赖
pnpm install

# 开发模式
pnpm dev

# 生产模式（推荐局域网/手机测试）
pnpm build && pnpm start
```

访问 http://localhost:3000

### 局域网访问（手机测试）

```bash
pnpm dev:lan
# 手机访问 http://<你的IP>:3000
```

> ⚠️ 摄像头扫码功能需要 HTTPS 环境。本地 `localhost` 可用，局域网 IP 需部署到 Vercel。

## 📐 数据流

```
问卷答案 (JSON ~2800 字符)
    │
    ▼ 自定义编码
编码字符串 (~860 字符)
    │  单选: "1:SM"  数值: "19:NZ" (Base36)
    │  多选: "11:MBABC" (位图)  文本: URI encode
    ▼ LZ-String 压缩
压缩字符串 (~650 字符)
    │
    ▼ QR Code 生成
二维码图片
```

## 📏 二维码容量

| 状态 | 范围 | 说明 |
|------|------|------|
| 🟢 安全 | < 1,500 字符 | 二维码清晰，秒扫 |
| 🟡 警告 | 1,500 ~ 2,200 字符 | 二维码较密，仍可扫 |
| 🔴 超限 | > 2,200 字符 | 可能生成失败或无法识别 |

实测 40 题全填满 → 压缩后约 1,300 字符，在安全范围内。

## 🎯 多选题编码策略

系统自动为每道多选题选择最短的编码方式：

| 策略 | 原理 | 适用场景 | 示例 |
|------|------|----------|------|
| 列表 | 直接拼接选项 ID | 选项少、选得少 | `MLABC` |
| 位图 | 二进制位 → Base36 | 选项多 (>6) | `MB15` |
| 组合映射 | 组合 → 索引号 | 组合总数 ≤ 1296 | `MC5` |

## 🆕 文件传输功能

### 工作原理

文件传输功能通过二维码序列传输任意类型的文件：

1. **发送端**：文件 → Base64 → LZ 压缩 → 分片 → 二维码序列
2. **接收端**：连续扫描 → 自动收集分片 → 重组 → 解压 → 下载

### 使用场景

- 🏥 医疗影像离线传输（X 光片、CT 等）
- 📄 文档资料交接（PDF、Word、Excel）
- 🎵 音频视频文件传输
- 📊 大型数据集离线同步
- 🔒 敏感数据的安全传输（不经过网络）

### 容量参考

| 文件大小 | 预估二维码数 | 传输时间 |
|---------|------------|---------|
| 100 KB  | ~60 个      | ~2 分钟  |
| 500 KB  | ~300 个     | ~10 分钟 |
| 1 MB    | ~600 个     | ~20 分钟 |

> 注：传输时间基于每 2 秒切换一个二维码，实际时间可能因网络环境和设备性能有所不同。

### 技术细节

- **分片大小**：1800 字符/片（确保二维码可扫）
- **压缩算法**：LZ-String（压缩率约 70%）
- **校验方式**：CRC32 完整性校验
- **编码格式**：Base64
- **去重机制**：自动跳过重复分片
