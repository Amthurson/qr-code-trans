## 1. 系统概述

### 这是什么？

一套 **完全离线** 的问卷数据传输系统：患者填写问卷后，答案被编码压缩生成二维码；医院扫码即可还原全部数据，**全程无需联网**。

### 核心流程

```
┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│  患者填写  │ ──→ │ 编码压缩  │ ──→ │ 生成二维码 │ ──→ │ 医院扫码  │
│  问卷答案  │     │ LZ-String │     │  QR Code  │     │ 解码还原  │
└──────────┘     └──────────┘     └──────────┘     └──────────┘
```

### 支持的题型

| 题型 | 图标 | 编码方式 | 示例 |
|------|------|----------|------|
| 单选题 | ○ | 选项 ID（1 字符）| 性别：男/女 |
| 多选题 | ☑ | 列表/位图/组合映射 | 症状多选 |
| 数值题 | 123 | Base36 编码 | 年龄、身高 |
| 填空题 | ✏️ | URI 编码 | 诊断名称 |
| 问答题 | 📝 | URI 编码 | 病情描述 |

## 2. 快速开始

### 本地启动

```bash
# 1. 进入项目目录
cd offline-questionnaire-qrcode

# 2. 安装依赖
pnpm install

# 3. 开发模式启动
pnpm dev

# 4. 或生产模式（推荐局域网测试）
pnpm build && pnpm start
```

### 局域网访问（手机测试）

```bash
# 局域网模式启动
pnpm dev:lan
# 手机访问 http://<你的IP>:3000
```

> ⚠️ **注意**：手机摄像头扫码功能需要 HTTPS 环境，建议部署到 Vercel。

详见 [DEPLOY.md](./DEPLOY.md)

## 4. 患者端 — 填写问卷并生成二维码

### 4.1 进入患者端

点击首页 **「👤 患者端」** 卡片，或直接访问 `/patient`。

### 4.2 填写基本信息

页面顶部有三个必填字段：

```
┌───────────────────────────────────────┐
│ 📝 基本信息                            │
│                                       │
│  患者 ID *      姓名 *       年龄 *    │
│  ┌──────────┐  ┌──────────┐ ┌──────┐  │
│  │ P12345   │  │ 张三     │ │ 35   │  │
│  └──────────┘  └──────────┘ └──────┘  │
└───────────────────────────────────────┘
```

![](http://cdn.shuyuancloud.com/image-20260325101850126.png)

- **患者 ID**：编号（如 P12345），用于识别
- **姓名**：患者姓名
- **年龄**：数字

> 💡 这三项都是必填的，不填无法生成二维码。

### 4.3 逐题回答问卷

向下滚动，依次回答每道题目。标有 **红色星号 (*)** 的是必填题。

#### 单选题

```
┌───────────────────────────────────────┐

│                                       │

└───────────────────────────────────────┘
```

点击选项即可选择，同一题只能选一个。

#### 多选题

```
┌───────────────────────────────────────┐
│ 6. 以下哪些症状您有？（可多选）          │
│                                       │
│  ☑ 双肩不等高                          │
│  ☑ 背部不对称                          │
│  ☐ 腰部疼痛                           │
│  ☑ 容易疲劳                           │
```

- 注意右下角 **最大可选数** 限制
- 达到上限后无法继续勾选

## 8. 常见问题 FAQ

### Q: 摄像头权限被拒绝怎么办？

**浏览器地址栏** 左侧有个 🔒 图标，点击它：
- 找到「摄像头」权限
- 改为「允许」
- 刷新页面重试

### Q: 手机上摄像头无法启动？

- 确认使用 **HTTPS** 访问（Vercel 部署或 `localhost`）
- 确认没有其他 App 占用摄像头
- 尝试切换前置/后置摄像头
- 如果仍然不行，使用 **「🖼️ 上传图片」** 方式

### Q: 二维码太密，扫不出来？

说明数据量太大。解决方法：
- 减少问答题的文字长度
- 减少选填项的填写
- 查看体积信息，确保在 🟢 安全范围内（< 1500 字符）

### Q: 扫码后只显示部分题目？

~~之前的版本中医院端只识别 10 题基础问卷模板。~~ **已修复（v1.0）**，现在会自动匹配问卷模板，40 题压力测试二维码也能完整解码。

### Q: 局域网 IP 访问页面空白/按钮无反应？

Next.js 开发模式的安全限制。解决方法：
1. 在 `next.config.ts` 中添加 `allowedDevOrigins: ['你的IP']`
2. 或使用生产模式：`pnpm build && pnpm start -- -H 0.0.0.0`

### Q: 如何新增问卷模板？

1. 在 `src/lib/` 下创建新的问卷文件（参考 `questions.ts`）
2. 在 `src/lib/questions.ts` 的 `questionnaires` 数组中添加
3. 医院端会自动识别新模板

## 9. 技术架构简介

### 数据流

```
问卷答案 (JSON)
    │
    ▼
自定义编码 (encoder.ts)
    │  单选: "1:SM"  多选: "11:MBABC"  数值: "19:NZ" (Base36)
    │  文本: "29:T%E9%9D%92%E5%B0%91" (URI encode)
    ▼
管道分隔拼接: "V2|P:P12345|A:35|S:M|1:SM|2:NZ|..."
    │
    ▼
LZ-String 压缩 (compressor.ts)
    │  平均压缩率 70-80%
    ▼
QR Code 生成 (qrcode 库)
    │  纠错级别: M (标准) / L (极限测试)
    ▼
二维码图片
```

### 文件结构

```
src/
├── app/

```

### 多选题三种编码策略

| 策略 | 原理 | 示例 | 适用场景 |
|------|------|------|----------|
| **列表 (list)** | 直接拼接选项 ID | `MLABC` → A,B,C | 选项少、选得少 |
| **位图 (bitmap)** | 二进制位表示 → Base36 | `MB15` → A,C,E | 选项多 (>6) |
| **组合 (combo)** | 组合映射到索引 | `MC5` → 第 5 种组合 | 组合总数 ≤ 1296 |

系统自动选择最短的策略。
